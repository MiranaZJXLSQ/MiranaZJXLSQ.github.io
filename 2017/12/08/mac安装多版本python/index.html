mac安装多版本python

发表于 2017-12-08

#mac上安装python

mac系统是自带的python的，但是mac系统里面有很多系统工具依赖于python，修改了python的版本，很多依赖python的系统工具也无法使用。开始的时候我看了别人的博客，想采用pyenv这种方式在管理python版本。

使用brew安装pyenv

默认你装好了brew
brew install pyenv
然后加入环境变量
$echo 'export PYENV_ROOT="$HOME/.pyenv"'>>~/.zshrc
$echo 'export PYENV_ROOT/bin:$PATH"'>>~/.zshrc
$echo 'eval "$(pyenv init -)"'>>~/.zshrc
$exec $SHELL -l
如果是你是bash的话或者是其他shell，可以把后面的配置文件的名字改成你想对应的shell的配置文件的名字。
三个写入文件中的语句的顺序是不能改变的，不然会导致”pyenv global/local”将不起作用。

使用pyenv安装python

查看可安装的python版本
pyenv install -l
选择好你要安装的版本
pyenv install <version>
安装完成之后对数据库进行更新
pyenv rehash
查看已经用pyenv安装的python版本
pyenv versions
切换python版本
pyenv global/local <version>
global会切换全局的python版本，而local命令会在当前目录下创建.python_version，管理当前目录及其子目录的python版本。
我用这种方法尝试了，但是始终下载不下来，速度炒鸡炒鸡炒鸡慢，可能是我比较着急，后来就换了一种不用pyenv的方法。

华丽的分割线———-
安装环境还是默认你已经装好了Xcode或者Xcode Command Line Tools以及Homebrew。
python --version
mac里面已经自带了python，为了安全，还是不要去乱搞，我们现在用homebrew来安装平时我们自己用的python。
homebrew查python
brew search python
你可以看到python 和 python3
安装python
brew install python
安装python3
brew install python3
这两个python会被安装在/usr/local/Cellar，如果你没有更改你的brew安装地址的话。
你可以看下文件夹里面的内容
open /usr/local/Cellar
或者简单展示下里面的内容
ls -al /usr/local/Cellar
设定python路径
echo $PATH
会看到/usr/bin:/usr/sbin:/sbin:/usr/local/bin等等你以前定义的环境变量
当你在shell里面输入brew的时候系统就会以此从第一个开始找，直到找到最后一个。
那么现在我们在/usr/bin里面有个系统python，而我们自己想要安装使用的python在/usr/local/Cellar里面
这样shell里面打开python的时候，就会找到系统的python
让我们来解决他~
sudo vim /etc/paths
我们只需要把/usr/local/bin放到最前面就可以了~
这时候我们在shell里面输入
which python
which python3
发现python3显示正常，而python~~
