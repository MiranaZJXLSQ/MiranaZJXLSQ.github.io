git学习笔记

12月初这段时间学习了一下git的一些基本命令。
git init–初始化一个git仓库，init会初始化一个repo，并且在当前路径下创建一个.git文件夹

###添加文件
把一个文件添加到git仓库只需要两步
git add <文件名字>
git commit -m “写一个readme文件帮助你记录”
-m不是必须的，但是应该对每次提交都有记录。
git commit -a --allow-empty-message -m ''
windows平台下可能需要变成这样
git commit -a --allow-empty-message -m ""

###查看工作区状态
git status 查询仓库当前的状态
-s 参数 -s表示short，-s的输出标记会有两列，第一列是对stage区域而言，第二列是对working目录
git diff 比较工作目录中当前文件和暂存区域快照之间的差异，也就是修改之后还没有暂存起来的变化的内容
–cached参数可以查看已经暂存起来的的文件和上次提交时的快照之间的差异

###查看版本历史记录
git log 查看当前版本的所有操作记录
–oneline 每条log只显示一行
–number 显示number条
git reflog 查看所有的版本操作记录

###版本回退命令
git reset
–hard –mixed –soft
soft(commit) < mixed(commit + add) < hard(commit + add + local working)
HEAD表示当前版本
HEAD^表示向前一个版本
HEAD^^表示向前两个版本
HEAD~100代表向前100个版本

###替换本地改动命令
git checkout -- <文件名字> 这个操作命令可以丢弃工作区，也就是本地更改

###推送当前分支命令
git push 把当前分支推送到远程
-u参数 git不但会把本地的master分支内容推送到远程新的master分支，还会把本地master分支和远程的master分支关联起来

###关联远程数据库命令
git remote add origin git@<server-name:path>/<repo-name>.git
关联后，使用命令git push -u origin master第一次推送master分支所有的内容
git支持多种协议，默认是ssh，也支持https，速度会稍微慢点。

###删除命令
git rm用于删除一个文件，如果一个文件已经被提交到版本库，不用担心误删，但是只能恢复到文件的最新版本

###分支系列命令
git branch可以用来列出分支，创建分支和删除分支
git branch -v 参数可以看到每一个分支的最后一次提交
git branch 列出本地所有分支，当前分支会被星号标出
git branch <分支的名字> 创建一个新的分支
git branch -d <分支的名字> 删除一个分支
git checkout可以用来切换分支
git checkout <分支的名字> 切换到一个分支
git checkout -b <分支的名字> 创建并切换到新的分支
git checkout -b <分支的名字> = git branch <分支的名字> + git chechout <分支的名字>
git merge 把一个分支合并进当前的分支
当git无法自动合并分支的时候，一定要先解决冲突，冲突解决后，再提交。

###显示分支合并图
git log --graph 分支合并图

###git的stash命令
git stash把当前的改动压入栈
git stash list 查看stash目录
git stash apply取出stash中的上一个项目（stash@{0}），并且应用于当前的工作目录
git stash drop删除上一个项目，也可以指定删除
git stash pop = git stash apply + git stash drop
git stash clear 删除所有项目

###git的标签系列命令
git tag <标签的名字>默认是HEAD 也可以指定一个commit id
-a -m “你想输入的关于此标签的信息”
-s 可以用gpg签名标签
git tag查看所有标签
git push origin <标签名字> 可以推送一个本地标签
git push origin –tags 可以推送全部未推送过的本地标签
git tag -d <标签的名字> 可以删除一个本地标签
git push origin :refs/tags/<标签的名字> 可以删除一个远程标签
